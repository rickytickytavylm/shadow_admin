# shadow_admin_panel

Админ-панель чемпионата **ТЕНЬ** — отдельный статический сайт (GitHub Pages).
Показывает заявки, полученные бэкендом `shadow_backend`.

Отдельный репозиторий, никак не связан с фронтом и бэком в одном проекте.

## Что делает

- Вход по токену (`ADMIN_TOKEN` из бэкенда), токен хранится в `localStorage`.
- Премиальный тёмный визуал в стиле основного сайта (glass, «ЧЕМПИОНАТ | ТЕНЬ»).
- **Заявки** — карточками; клик открывает панель со всей информацией:
  контакты, категория, ссылка на видео, комментарий, **Device ID**, IP, устройство.
- Смена **статуса** заявки: Новая / На рассмотрении / Прошёл отбор / Отклонена / Оплачено.
- **Ответ участнику на email прямо из панели** (если на бэкенде настроен SMTP) +
  история отправленных писем.
- **Чаты с ИИ** — список диалогов с Device ID пользователя и полной перепиской.
- Поиск и фильтры (по категории, статусу, тексту), обновление, выход.

## Настройка

Откройте `config.js` и укажите адрес бэкенда:

```js
window.SHADOW_ADMIN_CONFIG = {
  API_BASE: "https://<ваш-бэкенд>.up.railway.app",
};
```

Локально по умолчанию `http://localhost:8090`.

## Локальный запуск

Любой статический сервер, например:

```bash
cd shadow_admin_panel
npx serve .
# или: python -m http.server 5500
```

Затем откройте адрес в браузере и введите токен.

## Деплой на GitHub Pages

1. Отдельный репозиторий, запушить содержимое папки в ветку (например `main`).
2. Settings → Pages → Source: ветка `main`, папка `/root`.
3. Файл `.nojekyll` уже добавлен (чтобы Pages не трогал файлы).
4. В `config.js` указать боевой `API_BASE`.

## Важно про CORS

Домен админки должен быть добавлен в `ALLOWED_ORIGINS` бэкенда, иначе запросы
заблокирует CORS. Пример: `ALLOWED_ORIGINS=https://<user>.github.io`.

## Ответы участникам на email

Работает, если на бэкенде заданы переменные `SMTP_*` (см. `shadow_backend/.env.example`):

- `SMTP_HOST`, `SMTP_PORT`, `SMTP_SECURE`, `SMTP_USER`, `SMTP_PASS`
- `SMTP_FROM`, `SMTP_FROM_NAME`, `SMTP_REPLY_TO`

Подойдёт любой SMTP: Mail.ru, Яндекс, Gmail (пароль приложения). В панели открываете
заявку → пишете текст → «Отправить письмо». Участнику уходит письмо на указанный в
заявке email; его ответ приходит на `SMTP_REPLY_TO` (почту организатора).
Если SMTP не настроен — форма ответа показывает предупреждение и остаётся неактивной.

## Безопасность

Это лёгкая панель на токене — достаточно для приватного использования небольшой
командой. Токен даёт доступ только на чтение заявок. Не публикуйте токен и ссылку
на панель вместе. Для более строгой авторизации позже добавим полноценный логин.
